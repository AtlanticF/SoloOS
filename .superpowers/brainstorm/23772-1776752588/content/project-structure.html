项目结构方案

TS/Node 服务端 + React 客户端，两种组织方式

A

Monorepo（推荐）

pnpm workspace，server 和 web 各自独立，共享类型定义

SoloOS/
 ├── apps/
 │   ├── server/    # Hono + SQLite + AI
 │   │   ├── src/
 │   │   │   ├── routes/
 │   │   │   ├── db/
 │   │   │   └── skills/
 │   └── web/      # Vite + React
 │      └── src/
 │          ├── pages/
 │          └── components/
 ├── packages/
 │   └── shared/   # 共享 TS 类型
 └── package.json

优点

  • 类型在 server/web 间共享
  • 单命令启动全部
  • 易扩展（CLI 也加进来）

缺点

  • 初始配置多几行
B

双文件夹（极简）

根目录直接放 server/ 和 web/，各自独立 package.json

SoloOS/
 ├── server/
 │   ├── src/
 │   └── package.json
 ├── web/
 │   ├── src/
 │   └── package.json
 └── package.json # 只有启动脚本

优点

  • 结构最简单
  • 零配置启动

缺点

  • 类型需手动复制
  • 后期加 CLI 麻烦
推荐技术选型（方案 A）
SERVER
Hono — 轻量 TS-first，比 Express 快 3x
Drizzle ORM — SQLite，类型安全
Claude API — 异步分类
WEB
Vite + React — 开发体验最快
TanStack Query — 数据请求层
Recharts — 雷达图 + 趋势图
单命令启动：pnpm dev → server :3000 + web :5173 同时起来
