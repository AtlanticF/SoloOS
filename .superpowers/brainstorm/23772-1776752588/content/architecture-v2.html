系统架构 v2 — AI 分类层

分类机制作为开放接口，支持两种模式

AI 分类层设计
模式 1：API Key 模式
用户配置自己的 LLM API Key，Server 直接调用
# ~/.soloos/config.json
{
 "classifier": {
  "provider": "openai",
  "api_key": "sk-...",
  "model": "gpt-4o-mini"
 }
}
✓ 全自动，无需人工触发
✓ 支持任意 OpenAI-compatible API
✗ 用户需自行管理 API Key
模式 2：Agent Skill 模式
SoloOS 提供 Claude Code Skill，安装在用户 Agent 里定时执行
# Claude Code 中安装
/plugin install soloos-classifier

# Skill 内部逻辑：
GET /api/entries?status=pending
→ Claude 分类处理
→ PATCH /api/events (批量回写)
✓ 借用用户现有 Claude 会话
✓ 可定时 / 可手动触发
✗ 依赖用户运行 Claude Code
Server 侧的统一接口

不管哪种模式，Server 只暴露两个端点，分类结果从外部写入：

GET  /api/entries?status=pending   # 拉取待分类条目
POST /api/events/batch            # 批量写入分类结果

这样第三方也可以写自己的分类器接入，真正的开放接口。

完整数据流（v2）
CLI / Webhook
→ Entry 写入 DB (status: pending)
↓ 规则匹配（GitHub/Stripe → 直接成 Event）
↓ 无规则匹配 → 留在 pending 队列
分类器（模式1或2）
→ 拉取 pending → 分类 → 写回 Event
↓
Weekly Review
→ 用户确认/纠正 AI 的关联

这个架构准确吗？在终端确认，然后我们进入数据模型。
