SoloOS Design System

Inspired by Claude's warmth + Cursor's developer density — modern, minimal, data-first.

What We Take From Each

Claude Dashboard
Clean typographic hierarchy — information breathes
Neutral dark surface, not pitch black (#0d0d0d not #000)
Subtle borders (1px, low opacity) over heavy shadows
Generous micro-spacing inside cards
Muted accent colors — never neon
Cursor Dashboard
Developer-density: more information per pixel
Monospace for data values — creates "terminal" trust
Icon-only sidebar — maximizes canvas width
Command palette as primary navigation primitive
Status indicators: colored dots, not heavy badges

6 Core Design Rules

Rule 01 — Typography
Two-font system, strict roles
Inter for all UI copy. JetBrains Mono for all data values, timestamps, IDs, and metrics. Never mix roles.
Inter → labels, headings, body
JetBrains Mono → $2,400 · 14d · 0.92
Rule 02 — Color
Zinc scale + 5 pillar accents
Base palette is zinc (warm neutral). Each pillar owns one accent color used consistently — in badges, chart axes, and card borders.
■ Input   ■ Output
■ Audience  ■ Financial
■ Energy
Rule 03 — Density
Developer density, not consumer spacing
Padding inside cards: 14–20px. Row height: 32px. No wasted vertical space. Information should be scannable in one eye movement per card.
card-padding: 14px–20px
row-height: 32px
gap: 8px–12px (not 24px+)
Rule 04 — Motion
Purposeful only — no decorative animation
Only two animations: Sheet slide-in (180ms ease-out) and number counters on load. No hover float, no logo pulse. Motion means "something changed."
Sheet: 180ms ease-out
Number update: 300ms
Hover: opacity 0.7→1 only
Rule 05 — Interaction
Keyboard-first, command palette as spine
⌘K opens command palette from anywhere. Every action is reachable in ≤2 keystrokes. Mouse is optional. Inspired by Cursor's core interaction model.
⌘K → command palette
⌘C → capture entry
⌘R → start review
/ → filter current view
Rule 06 — Data Viz
Minimal chrome, maximum signal
Radar chart: no gridlines, filled polygon, pillar-colored axes. Sparklines: no axes, just the line. Every pixel serves data, not decoration.
Radar: filled + stroke, no grid
Sparklines: line only, no axes
Bar: 2px radius, no labels
